**用户角色**: 生产主管 / IE工程师

**痛点**: 每天要开MES系统，筛选产线、日期、指标，导出Excel做分析，耗时30-60分钟

**ManuGent 体验**:

```
用户: 3号线昨天白班和夜班的OEE对比一下

ManuGent:
📊 SMT-03 OEE 对比 (2026-04-15)
┌──────────┬────────┬────────┬────────┬────────┐
│ 班次     │ OEE    │ 可用率 │ 性能率 │ 良率   │
├──────────┼────────┼────────┼────────┼────────┤
│ 白班     │ 85.2%  │ 92.1%  │ 94.3%  │ 97.8%  │
│ 夜班     │ 72.6%  │ 88.5%  │ 85.1%  │ 96.1%  │
├──────────┼────────┼────────┼────────┼────────┤
│ 差异     │ -12.6% │ -3.6%  │ -9.2%  │ -1.7%  │
└──────────┴────────┴────────┴────────┴────────┘

🔍 主要差异原因:
1. 性能率差异最大 (-9.2%): 夜班22:00-01:00期间有3次换线，累计停机47分钟
2. 可用率差异 (-3.6%): 夜班02:30回流焊设备温度异常报警，停机18分钟
3. 良率差异较小: 两个班次均在正常范围内

💡 建议: 换线时间可通过提前备料优化，预计可提升夜班OEE 3-5%

**技术实现**:

- LLM对比分析 + 关联设备事件数据
- 返回结构化结果 + 自然语言解读

## 场景2: 良率异常根因分析

**用户角色**: 品质工程师

**痛点**: 良率突然下降，需要逐个排查设备、物料、人员、环境因素，可能要半天到一天

**ManuGent 体验**:

```
用户: 最近3天贴片良率从99.2%掉到了97.1%，帮我分析原因

ManuGent:
⚠️ 检测到 SMT-05 贴片良率异常下降

📈 趋势:
  04-12: 99.2% (正常)
  04-13: 98.5% ⚠️ 下降开始
  04-14: 97.8% ⚠️ 持续下降
  04-15: 97.1% ⚠️ 跌破警戒线

🔍 关联分析 (置信度 87%):

1. 【主因】锡膏印刷偏移 (贡献度 ~60%)
   - 04-13 起 SPI 检测到印刷偏移量增加 0.05mm
   - 关联事件: 04-12 夜班更换了刮刀，未做参数校准
   - 证据: 偏移方向一致，集中在 PCB-B 区域

2. 【次因】新批次物料 (贡献度 ~25%)
   - 04-13 切换了物料批次 LOT-20260412-A
   - 该批次焊盘尺寸公差比上批次大 0.02mm
   - 叠加印刷偏移导致焊接不良

3. 【可能】回流焊温度波动 (贡献度 ~15%)
   - 04-14 18:00 回流焊 Zone3 温度偏低 3°C
   - 但持续时间短，影响有限

📋 建议措施:
  1. 立即: 重新校准锡膏印刷机参数 (预计恢复1.5%良率)
  2. 短期: 与供应商确认新批次物料规格差异
  3. 长期: 刮刀更换后增加 SPI 首件确认流程

是否需要我创建一个品质异常处理单?
```

**技术实现**:
- RootCause Agent 多维关联分析:
  - 时间序列对比（良率曲线 × 设备参数曲线）
  - 事件关联（物料切换、设备维护、人员变动）
  - 同类产品对比（排除产品差异）
- 检索历史类似案例（RAG + 知识库）

## 场景5: 新员工培训助手

**用户角色**: 入职1周的新操作员

**痛点**: MES系统复杂，培训文档几百页，遇到问题不知道问谁

**ManuGent 体验**:

```
新员工: 这个报警是什么意思？设备显示 "FEEDER ERROR 0x4A"

ManuGent:
这是飞达(Feeder)供料器错误，代码 0x4A 表示"供料步进异常"。

📸 [显示设备报警界面照片 + 标注]

可能原因:
1. 料带卡住 — 最常见 (60%)
2. 飞达齿轮磨损 — 使用超过50万次后常见 (25%)
3. 传感器脏污 — 灰尘积累 (15%)

处理步骤 (请按顺序操作):
1. ⏸️ 按下设备暂停键 (黄色按钮)
2. 📂 打开飞达盖板
3. 👀 检查料带是否卡在齿轮处
4. 🔧 如果卡住: 轻轻拉出料带，重新装入
5. ▶️ 按复位键恢复运行
6. ❌ 如果仍报警: 通知设备工程师 (分机 8502)

⚠️ 安全提示: 操作前确保设备完全停止，不要在设备运行时打开飞达

📹 [播放标准操作视频]  📞 [一键呼叫工程师]
